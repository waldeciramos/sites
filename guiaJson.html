1 O problema original Problema

O sistema de ramais do Grupo Dom Pedro era uma página HTML única (ramais.html) que lia e gravava os dados em um arquivo ramais.json hospedado em um repositório do GitHub.

Para isso, o código continha um token de acesso pessoal do GitHub diretamente no HTML:

const GH_USER  = 'waldeciramos';
const GH_REPO  = 'dompedroprojeoes';
const GH_TOKEN = 'ghp_••••••••••••••••••••••••••••••••••••';
const GH_FILE  = 'ramais.json';
⚠️ O que deu errado: o GitHub detecta automaticamente tokens expostos em código publicado (mesmo em repositórios privados) e os revoga (invalida) automaticamente por segurança. Quando isso acontece, todas as chamadas à API do GitHub passam a falhar.

Sintoma observado

  • A página exibia ❌ Erro ao carregar ao tentar buscar o ramais.json
  • O sistema caía no modo de cache local (localStorage), que fica desatualizado
  • Alterações feitas por um usuário não eram vistas pelos demais
  • Mesmo quando funcionava, o GitHub tinha delay de cache (CDN) para refletir mudanças

2 Por que escolhemos o JSONBin Solução

Foram avaliadas 4 alternativas para substituir o GitHub:

  • Opção A — Backend próprio (Flask) como intermediário, guardando o token
  • Opção B — Google Sheets como banco de dados
  • Opção C — JSONBin.io ✅ (escolhida)
  • Opção D — GitHub com proxy (Cloudflare Worker)
Por que o JSONBin venceu: é um serviço gratuito de armazenamento de JSON na nuvem, com API simples baseada em chaves, sem necessidade de servidor próprio e com atualização quase instantânea (sem cache de CDN como o GitHub).

3 Criando a conta no JSONBin Configuração

Passos realizados para criar a conta gratuita:

  1. Acesse jsonbin.io
  2. Clique em Sign Up e crie a conta (e-mail + senha, ou login social)
  3. Confirme o e-mail, se solicitado
  4. Você será direcionado ao Dashboard em jsonbin.io/app/dashboard
📸 Tela do Dashboard após login — menu lateral com Dashboard, Bins, Collections, Schema Docs, API Keys, My Account

4 Gerando as chaves de API Configuração

  1. No menu lateral, clique em API Keys
  2. A chave X-MASTER-KEY já vem criada automaticamente com a conta — ela dá acesso total
  3. Para uma chave com permissões mais restritas, clique em + Create Access Key
  4. Dê um nome (foi usado: ramais) e confirme
  5. Copie as duas chaves geradas

Chaves utilizadas no projeto

X-MASTER-KEY $2a$10$lFTdWh/opLn8wg1UFKGyHe.YaOZINd4D2qnk2qGd0PFbv.Yz686Qm
X-ACCESS-KEY (ramais) $2a$10$eqC8ucHjOTUCijJGsw6YZeT16qAa6ybBCk2BmJdMwioIvoybhrBru
💡 Diferença: a X-MASTER-KEY tem acesso total à conta. A X-ACCESS-KEY pode ser configurada com permissões específicas para bins individuais — boa prática para não expor a master key em produção.

5 Criando o Bin com os dados Configuração

O Bin é o "documento" na nuvem onde o ramais.json fica armazenado.

  1. No menu lateral, clique em Bins
  2. Inicialmente aparece No Bins Found — nenhum bin criado ainda
  3. Clique em Create a Bin
  4. Apague o JSON de exemplo (placeholder com users, firstName etc.)
  5. Cole o conteúdo completo do arquivo ramais.json do Grupo Dom Pedro
  6. Clique em Create / Save
Resultado: o sistema gerou automaticamente o Bin ID: 6a2ac3d9da38895dfeadf3b3
Visibilidade: Private 🔒 (protegido pelas chaves de API)

Depois (JSONBin — funcionando)

const JB_MASTER_KEY = '$2a$10$lFTdWh/opLn8wg1UFKGyHe.YaOZINd4D2qnk2qGd0PFbv.Yz686Qm';
const JB_ACCESS_KEY = '$2a$10$eqC8ucHjOTUCijJGsw6YZeT16qAa6ybBCk2BmJdMwioIvoybhrBru';
const JB_BIN_ID     = '6a2ac3d9da38895dfeadf3b3';
const JB_API        = `https://api.jsonbin.io/v3/b/${JB_BIN_ID}`;

Função de leitura (GET)

Busca a versão mais recente do bin usando o endpoint /latest:

const res = await fetch(JB_API + '/latest', {
  headers: {
    'X-Master-Key': JB_MASTER_KEY,
    'X-Access-Key': JB_ACCESS_KEY
  }
});
const json = await res.json();
DATA = json.record; // conteúdo do ramais.json

Função de gravação (PUT)

Sobrescreve o conteúdo do bin com os dados atualizados:

const res = await fetch(JB_API, {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
    'X-Master-Key': JB_MASTER_KEY,
    'X-Access-Key': JB_ACCESS_KEY
  },
  body: JSON.stringify(DATA)
});
Fallback offline: em caso de falha de conexão, o sistema mantém uma cópia em localStorage (ramais_fallback) e a exibe com aviso ⚠️ Usando cache local.
