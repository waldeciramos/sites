🔧 1. Configuração do Samba (Base + Acesso para WinPE)

Esta etapa une seu manual Samba existente com os compartilhamentos extras necessários para o netboot.xyz. Toda a instalação e configuração do Samba fica otimizada.

1 Remover qualquer instalação anterior

sudo systemctl stop smbd nmbd 2>/dev/null
sudo apt-get remove --purge -y samba samba-common smbclient
sudo rm -rf /etc/samba /var/lib/samba /var/log/samba /srv/samba /home/waldeci/samba-share /opt/netboot

2 Instalar Samba + Utilitários (inclui docker para netboot)

sudo apt-get update -y
sudo apt-get install -y samba smbclient docker.io docker-compose rsync wget
sudo systemctl enable --now docker

3 Criar estrutura de pastas (Samba pessoal + pasta wininstall)

sudo mkdir -p /srv/samba/publico
sudo mkdir -p /home/waldeci/samba-share
sudo mkdir -p /opt/netboot/samba/windows_iso
sudo mkdir -p /opt/netboot/samba/configs
sudo mkdir -p /opt/netboot/assets/WinPE/x64
sudo chmod 777 /srv/samba/publico
sudo chown waldeci:waldeci /home/waldeci/samba-share
sudo chmod 755 /home/waldeci/samba-share
sudo chmod -R 775 /opt/netboot/samba

4 Configuração unificada do smb.conf (pastas + wininstall)

sudo tee /etc/samba/smb.conf << 'EOF'
[global]
workgroup = WORKGROUP
server string = Servidor Samba + PXE
security = user
map to guest = Bad User
guest account = nobody
log file = /var/log/samba/log.%m
max log size = 1000

[publico]
path = /srv/samba/publico
browseable = yes
read only = no
guest ok = yes
writable = yes
create mask = 0777
directory mask = 0777

[waldeci]
path = /home/waldeci/samba-share
browseable = yes
read only = no
guest ok = no
valid users = waldeci
create mask = 0755
directory mask = 0755

[wininstall]
path = /opt/netboot/samba
browseable = yes
read only = yes
guest ok = yes
force user = nobody
; Para maior compatibilidade com Windows PE, permitimos guest
EOF

5 Criar usuário waldeci no Samba (senha 1)

sudo smbpasswd -a waldeci
# Digite a senha: 1 (duas vezes)
sudo smbpasswd -e waldeci
sudo systemctl restart smbd
sudo systemctl enable smbd
✅ Samba ativo com três compartilhamentos: publico, waldeci e wininstall (para instalação Windows).

🐳 2. netboot.xyz - Servidor PXE/iPXE (Docker)

Usamos a imagem oficial do netboot.xyz para servir o boot via rede (HTTP/TFTP). O cliente fará PXE, carregará o menu e depois o WinPE montará o compartilhamento SMB.

1 Baixar e executar contêiner netboot.xyz

docker run -d \
  --name netbootxyz \
  --restart unless-stopped \
  -p 80:80 \
  -p 69:69/udp \
  -p 4011:4011/udp \
  -v /opt/netboot/assets:/config/assets \
  -v /opt/netboot/config:/config \
  -e NGINX_PORT=80 \
  ghcr.io/netbootxyz/netbootxyz:latest
🔍 Portas: 80 (web/assets), 69/udp (TFTP), 4011 (proxyDHCP). Certifique-se que não há conflito com outros serviços web.

2 Ajustar permissões e preparar WinPE customizado

Extraia o conteúdo da sua ISO do Windows dentro da pasta /opt/netboot/samba/windows_iso — assim o Samba compartilhará os arquivos de instalação.

# Exemplo de extração da ISO (substitua pelo caminho da sua ISO)
sudo mount -o loop /caminho/para/Windows.iso /mnt/win_iso
sudo rsync -av /mnt/win_iso/ /opt/netboot/samba/windows_iso/
sudo umount /mnt/win_iso

# Agora copie o boot.wim para o diretório de assets do netboot (WinPE boot)
sudo cp /opt/netboot/samba/windows_iso/sources/boot.wim /opt/netboot/assets/WinPE/x64/boot.wim

3 Criar script de automação para WinPE (montar SMB automaticamente)

Dentro da pasta de configurações, criamos winpeshl.ini e configure.bat para que o Windows PE monte o share e execute o setup.exe sozinho.

sudo mkdir -p /opt/netboot/assets/configs/winpe_mount
sudo tee /opt/netboot/assets/configs/winpe_mount/winpeshl.ini << 'EOF'
[LaunchApp]
AppPath = X:\Windows\System32\configure.bat
EOF

sudo tee /opt/netboot/assets/configs/winpe_mount/configure.bat << 'EOF'
@echo off
wpeinit
timeout /t 3 /nobreak > nul
:loop
ping -n 1 192.168.2.156 > nul
if errorlevel 1 goto loop
net use Z: \\192.168.2.156\wininstall /user:guest ""
Z:\windows_iso\setup.exe
exit
EOF
⚠️ Substitua o IP 192.168.2.156 pelo IP real do seu servidor. Caso use autenticação, ajuste o net use com usuário/senha (ex: waldeci/1).

4 Customizar menu netboot.xyz para apontar ao WinPE local

Crie um arquivo de override local-vars.ipxe para que o netboot.xyz sirva o WinPE com os scripts.

sudo tee /opt/netboot/config/assets/local-vars.ipxe << 'EOF'
# Configuração para Windows PE customizado
set win_base_url http://192.168.2.80/WinPE
set win_pe_boot boot.wim
echo netboot.xyz + Samba integrado: WinPE pronto
EOF
# Ajuste o IP para o IP do servidor (porta 80 é a do container)
# Em alguns casos, use http://SEU_IP/WinPE/boot.wim
sudo tee /opt/netboot/config/assets/menu/win-custom.ipxe << 'EOM'
#!ipxe
goto win_custom_chain
:win_custom_chain
kernel wimboot
initrd ${win_base_url}/boot.wim boot.wim
boot
EOM
Para usar diretamente via menu, após a inicialização do netboot.xyz acesse "Windows Installer" e coloque o URL. Mas a abordagem acima já deixa os assets acessíveis.

🧪 4. Verificação e boot do cliente

✅ Testes locais no Linux

# Testar acesso ao Samba wininstall

# Testar boot.wim acessível via HTTP

🖥️ Cliente Windows: boot PXE

1. Na BIOS/UEFI ative o boot pela rede (PXE).
2. O cliente receberá IP do DHCP (certifique-se que o servidor DHCP aponta o next-server para o IP do seu Ubuntu, porta 4011 se necessário).
3. Menu netboot.xyz aparecerá → escolha "Windows" → ambiente WinPE.
4. O script configure.bat montará \\192.168.2.156\wininstall e iniciará o setup.exe automático.
✔️ Instalação fluirá sem intervenção.

Dica DHCP: Caso seu roteador não permita opção 66, você pode usar o dnsmasq em modo proxy. Exemplo rápido:
sudo apt install dnsmasq -y && sudo tee /etc/dnsmasq.d/netboot.conf <<< "dhcp-range=192.168.2.100,192.168.2.200,12h\ndhcp-option=66,192.168.2.156\ndhcp-option=67,undionly.kpxe" && sudo systemctl restart dnsmasq

⚡ Comando Unificado (Configuração total em um bloco)

Copie e execute este comando gigante para configurar Samba + netboot.xyz + estrutura WinPE de uma só vez (substitua IP se necessário).

sudo bash -c 'set -e; apt update; apt install -y samba docker.io rsync; systemctl enable --now docker; mkdir -p /opt/netboot/{assets/{WinPE/x64,configs/winpe_mount},samba/windows_iso}; mkdir -p /srv/samba/publico /home/waldeci/samba-share; chmod 777 /srv/samba/publico; chown waldeci:waldeci /home/waldeci/samba-share; tee /etc/samba/smb.conf << "SMBEOF"
[global]
workgroup=WORKGROUP
map to guest=Bad User
[publico]
path=/srv/samba/publico
guest ok=yes
read only=no
[waldeci]
path=/home/waldeci/samba-share
valid users=waldeci
[wininstall]
path=/opt/netboot/samba
guest ok=yes
read only=yes
SMBEOF
echo -e "1\n1" | smbpasswd -a waldeci -s; systemctl restart smbd; docker run -d --name netbootxyz --restart unless-stopped -p 80:80 -p 69:69/udp -p 4011:4011/udp -v /opt/netboot/assets:/config/assets -v /opt/netboot/config:/config ghcr.io/netbootxyz/netbootxyz:latest; echo "Sistema pronto!"'
Após o comando, não se esqueça de extrair a ISO do Windows dentro de /opt/netboot/samba/windows_iso e copiar o boot.wim para assets.

🪟 Acesso aos Compartilhamentos (Windows Cliente)

1. No explorador de arquivos digite: \\192.168.2.156
2. Pastas disponíveis: publico (acesso livre), waldeci (senha: 1), wininstall (onde está a ISO extraída).
3. Para testar se o caminho está correto para o setup do Windows: dentro da rede o cliente verá \\192.168.2.156\wininstall\windows_iso\setup.exe.

Limpar credenciais antigas no Windows: net use * /delete e cmdkey /delete:192.168.2.156.

🔧 Troubleshooting integrado (Samba + netboot)

❌ Cliente não faz PXE: Verifique se o firewall liberou UDP 67,68,69,4011. Execute sudo ufw allow 69/udp; sudo ufw allow 4011/udp; sudo ufw allow 80/tcp.
❌ WinPE não enxerga o compartilhamento wininstall: Teste manualmente no WinPE pressionando Shift+F10, digite net use Z: \\192.168.2.156\wininstall. Se der erro, confira permissões e firewall SMB (portas 139,445 liberadas).
❌ Setup não encontra disco (VMware/VirtualBox): Injete drivers virtIO no boot.wim usando DISM: dism /Mount-Image /ImageFile:boot.wim /MountDir:./mount e adicione os drivers.
✔️ Para testar o netboot.xyz sem PXE, use a ISO bootável: Baixar netboot.xyz.iso e boot via pendrive — útil para testes.
